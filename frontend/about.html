SYSTEM ARCHITECTURE

The Compute Constraint:
While deploying an image classifier is straightforward with access to heavy GPU clusters, this project was designed around artificial compute constraints. Drawing from enterprise-grade system configuration, my goal was to architect a microservices pipeline capable of running locally on a standard CPU while maintaining production-level accuracy.

1. Spatial Caching via C++:
Because neural network inference is computationally expensive, I engineered a native C++ layer to intercept requests and hash images at the byte level. By evaluating these hashes against a PostGIS spatial database, the system forms a high-speed caching layer, shielding the PyTorch models from duplicate processing.

2. Quantized Edge Compute (PyTorch):
When a cache miss occurs, the payload is forwarded to a Python FastAPI endpoint running a PyTorch vision model. To maintain the optimization constraint, it utilizes a lightweight MobileNetV3 backbone. By aggressively pruning weights and applying INT8 Dynamic Quantization (compressing 32-bit floating-point parameters down to 8-bit integers), the model achieves near-instant inference with significantly reduced memory utilization.
