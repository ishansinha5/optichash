SYSTEM ARCHITECTURE

The "Optimized AI" Constraint:
It is easy to build an image classifier if you have unlimited cloud credits and heavy GPU clusters. This project was an exercise in artificial constraints. I wanted to build a B2B-style microservice architecture capable of running entirely on constrained edge hardware (or a standard laptop CPU) while maintaining high accuracy.

1. Spatial Caching via C++:
Machine learning is expensive; simple math is cheap. I built a native C++ layer that intercepts requests and hashes images at the byte level. By checking these hashes against a PostGIS spatial database, the system creates a high-speed caching layer that shields the heavier PyTorch models from duplicate traffic.

2. Quantized Edge Compute (PyTorch):
When the C++ cache misses, the request is handled by a Python FastAPI endpoint running a vision model. Instead of a massive architecture, it uses a lightweight MobileNetV3 backbone. By aggressively pruning the weights and applying INT8 Dynamic Quantization (compressing 32-bit floating-point math down to 8-bit integers), the model executes near-instant inference with a fraction of standard RAM utilization.
